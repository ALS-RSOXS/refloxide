# Reference data for regression tests

Golden data files in this directory are produced by running the two
published reference implementations of the Passler-Paarmann 4x4 formalism
against the canonical test stacks defined in
[`tests/regression/_stacks.py`](../_stacks.py). The files are not checked
in. Users running the regression suite locally must regenerate the
golden files themselves, or obtain them from the project maintainers.

## Layout

```
references/
├── README.md
├── passler_matlab/
│   └── sic_gan_sic_otto.npz     (generated by generate_matlab.py)
└── jeannin_python/
    ├── sic_gan_sic_otto.npz     (generated by generate_jeannin.py)
    └── uniaxial_substrate.npz   (generated by generate_jeannin.py)
```

## Passler MATLAB reference

The Passler MATLAB reference is published at
[Zenodo 601496](https://doi.org/10.5281/zenodo.601496) and was updated
alongside the 2019 erratum. To generate the golden data you need either
MATLAB or GNU Octave. The generator script lives at
[`generate_matlab.py`](generate_matlab.py) and calls out to MATLAB via
subprocess.

Expected array layout of the generated `.npz` file:

| key                 | dtype      | shape            |
|---------------------|------------|------------------|
| `theta_rad`         | float64    | `(N_theta,)`     |
| `wavenumber_cm`     | float64    | `(N_k,)`         |
| `thickness_gan_nm`  | float64    | scalar           |
| `r_pp`, `r_ss`, ... | complex128 | `(N_theta, N_k)` |

The eight amplitudes use the Passler-Paarmann naming described in
[`docs/theory/reflection_transmission.md`](../../../docs/theory/reflection_transmission.md).

## Jeannin Python reference

The Jeannin Python port is published at
[Zenodo 3417751](https://doi.org/10.5281/zenodo.3417751). Unlike the
MATLAB reference, the Jeannin port can be invoked directly from Python
and the generator script [`generate_jeannin.py`](generate_jeannin.py)
calls it as a library. Install the port into the project virtual
environment before running the generator.

Expected array layout of `sic_gan_sic_otto.npz` matches the MATLAB
reference. The `uniaxial_substrate.npz` file has a single wavenumber
and a one-dimensional sweep in `theta_rad`.

## Regeneration workflow

1. Obtain a local working copy of either reference implementation.
2. Point the generator script at the reference source.
3. Run the generator. Golden `.npz` files appear in the appropriate
   subdirectory.
4. Re-run `pytest tests/regression/ -m regression`. Tests previously
   skipping on missing references should now run.

## Notes on tolerances

The `tol_reference_code` fixture in
[`conftest.py`](../conftest.py) sets the agreement tolerance at `1e-8`
for comparisons against either reference. This tolerance accounts for
the different matrix-inversion strategies used by MATLAB (Gauss-Jordan
on condition-normalized matrices) versus the eventual refloxide kernel
(LU with partial pivoting in `nalgebra` or equivalent). Tightening the
tolerance further is appropriate only once the kernel exposes its own
matrix-inversion precision knob.
